1. Asymptotic Notations

Big O ( O )

Upper Bound. "The algorithm grows NO FASTER than this."

Formal: f(n) ≤ c · g(n) for all n ≥ n∞

O(N^2) covers N, N log N, and N^2.

Big Θ ( Θ )

Tight Bound. "The algorithm grows EXACTLY like this."

Formal: c1·g(n) ≤ f(n) ≤ c2·g(n)

Θ(N) means strictly linear. No faster, no slower.

Big Ω ( Ω )

Lower Bound. "The algorithm takes AT LEAST this long."

Formal: f(n) ≥ c · g(n) for all n ≥ n∞

Ω(1) means it takes at least 1 operation.

3. Amortized Complexity

Amortized analysis averages the time required to perform a sequence of data-structure operations. Even if a single operation is expensive (like resizing), the average cost over a sequence is low.

Case Study: std::vector::push_back

When a vector is full, adding an element requires allocating a new array of double the capacity, copying old elements, and adding the new one. This specific resize is O(N).

Cap: 1 → Cap: 2 → Cap: 4 → Cap: 8 → Cap: 16

Let's insert N elements. The cost is 1 for each insert, plus the cost of copying during resizes. The copies happen at sizes 1, 2, 4, 8... up to N.

Total Copy Cost = 1 + 2 + 4 + 8 + ... + N/2 < N.
Total Overall Cost = N (inserts) + N (copies) = 2N.
Average cost per operation = 2N / N = O(1) Amortized.

4. Loop Analysis Patterns

// Simple Loop -> O(n)
for(int i=0; i<n; i++) { ... }

// Nested Loops -> O(n^2)
for(int i=0; i<n; i++) {
    for(int j=0; j<n; j++) { ... }
}

// Triangular Loops -> O(n^2)
for(int i=0; i<n; i++) {
    for(int j=0; j<i; j++) { ... } // Operations = n*(n-1)/2
}

// Logarithmic Loops -> O(log n)
for(int i=1; i<n; i*=2) { ... } // i doubles each step

Block 3: Harder Analysis (4 Hours)

Harmonic series, nested logs, mixed patterns, and recurrence relations.

Harmonic Complexity

for(int i=1; i<=n; i++) {
    for(int j=1; j<=n; j+=i) {
        // ...
    }
}

The inner loop runs n/i times.
Sum = n/1 + n/2 + n/3 + ... + n/n
Sum = n * (1 + 1/2 + 1/3 + ... + 1/n)
The sum of the harmonic series grows logarithmically.
Total Complexity: O(n log n). Extremely common in number theory (Sieve variants).

Logarithmic Nested Loops

for(int i=1; i<n; i*=2) {
    for(int j=0; j<i; j++) {
        // ...
    }
}

The inner loop bounds depend on i, which doubles: 1, 2, 4, 8...
Operations: 1 + 2 + 4 + 8 + ... + n
This is a geometric series. The sum is 2n - 1.
Total Complexity: O(n). Do not just multiply loop depths!

Square Root Loops

for(int i=1; i*i<=n; i++) {
    // Prime factorization, divisors, etc.
}

The loop terminates when i^2 > n, meaning i > √n. Thus, the loop runs exactly √n times. Complexity: O(√n).

Task 1: The CP Benchmark Template

Always use ios::sync_with_stdio(false). Use chrono for precision timing.

#include <iostream>
#include <vector>
#include <chrono>
using namespace std;
using namespace std::chrono;

void solve(int n) {
    // Your O(n), O(n^2), etc. code here
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);
    
    int n = 100000;
    auto start = high_resolution_clock::now();
    
    solve(n);
    
    auto stop = high_resolution_clock::now();
    auto duration = duration_cast<milliseconds>(stop - start);
    cout << "Time: " << duration.count() << " ms\n";
    return 0;
}

Reflection & Checklist

  • ✓ Identify N: What is the primary input size driving the loop/recursion?
  • ✓ Nested != Multiplication: Are inner loops dependent on outer loop variables?
  • ✓ Logarithmic Steps: Are loop variables multiplying or dividing by constants?
  • ✓ Worst Case: What happens if the `break` statement is never hit?
  • X Common Mistake: Assuming string concatenation `s += "a"` is O(1) in all languages. (In C++ `std::string` it is amortized O(1), but `s = s + "a"` is O(N)).
