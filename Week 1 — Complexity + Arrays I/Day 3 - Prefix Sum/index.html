Block 1: Architectural Design

Transitioning from linear-time $O(n)$ sequence scanning to constant-time $O(1)$ query aggregation.

1. Inclusion-Exclusion Formulas

Sum(L, R) = P[R] - P[L-1]

Precompute cumulative totals so that the internal dependencies of a sequence are mathematically resolved. Any arbitrary segment is isolated and evaluated through strict algebraic subtraction of unwanted prefixes.

2. Scan Operator Inverses

Any associative operator with an inverse allows for O(1) range queries.

a ⊕ b → P[R] ⊕ P[L-1]
(XOR is its own inverse)
max(L, R) → P[R] - P[L-1]
(Max has no inverse!)

3. Spatial Scaling (2D Matrix sums)

The 2D Formula: Sum(r1, c1, r2, c2) = P[r2][c2] - P[r1-1][c2] - P[r2][c1-1] + P[r1-1][c1-1]

This technique (The Integral Image) powers early real-time Computer Vision like Haar-Cascade face detection (Viola-Jones).

Block 3: The Map Trick ($O(N^2) \rightarrow O(N)$)

Combining cumulative parity and hashing.

P[i] - P[j] = K

When scanning an array looking for a dynamic condition $K$ (e.g. subarray sum equals K), we mathematically transform the request from a 2-pointer sliding window into a historic lookup.

Since: P[i] - P[j] = K
Then: P[j] = P[i] - K

As you iterate and generate $P[i]$, you simultaneously query a Hash Map for the existence of $P[i] - K$. If found, you immediately know a subarray exists ending at your current index.
